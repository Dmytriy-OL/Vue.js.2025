# Лабораторна робота 3: Основи стилізації та анімації у Vue.js

Простий новинний сайт ("NewsMini"), що демонструє різні підходи до стилізації компонентів, використання CSS Modules, Scoped Styles, а також роботу з анімаціями та Teleport.

## Інструкція запуску

1. **Встановлення залежностей**:
   ```bash
   npm install
   ```
2. **Запуск сервера розробки**:
   ```bash
   npm run dev
   ```
3. Відкрийте у браузері посилання, яке з'явиться в терміналі (зазвичай `http://localhost:5174`).

## Висновки про особливості стилізації у Vue.js

У ході виконання роботи було досліджено та використано три основні підходи до стилізації у Vue.js, кожен з яких має свої переваги для конкретних задач:

1.  **Global Styles (Глобальні стилі)**
    *   **Реалізація**: Імпорт файлу `base.css` у `main.js`.
    *   **Призначення**: Визначення CSS-змінних (кольорів, шрифтів), скидання стилів (reset CSS) та базових налаштувань для всього проєкту.
    *   **Перевага**: Доступність змінних (наприклад, `var(--primary)`) у будь-якому компоненті.

2.  **Scoped Styles (Ізольовані стилі)**
    *   **Реалізація**: Тег `<style scoped>` у компоненті (наприклад, у `SiteHeader.vue`, `App.vue`).
    *   **Призначення**: Стилізація конкретного компонента без ризику вплинути на інші частини застосунку. Vue автоматично додає унікальні атрибути (`data-v-xxxx`) до елементів.
    *   **Перевага**: Простота використання та гарантія того, що стилі "хедеру" не зламають "футер".

3.  **CSS Modules**
    *   **Реалізація**: Файли з розширенням `.module.css` (наприклад, `AdModal.module.css` для `AdModal.vue`).
    *   **Призначення**: Використання класів як об'єктів JavaScript (`:class="styles.modal"`).
    *   **Перевага**: Повна ізоляція імен класів (генеруються унікальні хеші), що дозволяє використовувати прості назви типу `.modal`, `.title` без конфліктів, навіть якщо такі класи є в глобальних стилях.

### Додаткові особливості:
*   **Vue Transitions**: Вбудований компонент `<Transition>` (використаний у `App.vue` для переходів між сторінками та в `AdModal.vue` для появи модалки) дозволяє легко додавати анімації входу/виходу, просто описуючи CSS-класи станів (наприклад, `.modal-enter-active`).
*   **Teleport**: Дозволяє рендерити компонент (модалку) в іншому місці DOM-дерева (наприклад, у `body`), зберігаючи при цьому логічний зв'язок з батьківським компонентом. Це критично важливо для правильного позиціювання перекриваючих елементів (overlays).
